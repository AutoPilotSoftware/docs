# 🍎 Запуск AutoPilot на macOS

Повна інструкція для першого запуску AutoPilot на Mac (Apple Silicon / Intel).

> 💡 **Коротка версія** є в розділі [FAQ → Запуск на macOS](/docs/ua/faq/#16--запуск-на-macos). Ця сторінка — повний гайд з усіма troubleshooting-випадками.

---

## 📥 Де завантажити збірку для macOS

Актуальні збірки публікуються у **AutoPilot Chat** (Telegram) в окремих гілках по біржах:

| Гілка чату | Біржа |
|------------|-------|
| **Updates Bybit** | Bybit AutoPilot |
| **Updates MEXC** | MEXC AutoPilot |
| **Updates Bitget** | Bitget AutoPilot |

Відкрийте потрібну гілку → завантажте останню збірку з позначкою **macOS** → розпакуйте архів у зручну папку (наприклад, `~/Downloads/AutoPilot/`) → переходьте до кроків нижче.

> 💡 **Apple Silicon vs Intel:** зазвичай у чаті викладаються обидві збірки — `darwin-arm64` для M1/M2/M3/M4 та `darwin-x64` для Intel. Як визначити свій чіп — див. розділ troubleshooting нижче.

> 📖 **Про автооновлення** всередині софту — див. [FAQ → Оновлення](/docs/ua/faq/#12--оновлення).

## Чому це складніше, ніж на Windows

macOS за замовчуванням блокує запуск непідписаних бінарних файлів, завантажених з інтернету:

- Файл `AutoPilot` у Finder виглядає як «документ», а не як програма
- Система поміщає його у **карантин Gatekeeper** — подвійний клік не працює
- Може з'явитися помилка `cannot be opened because the developer cannot be verified`
- Може з'явитися помилка `killed: 9` при спробі запуску

Це вирішується двома командами в терміналі. Один раз — і далі запуск простий.

## Рекомендований термінал

Найкраще використовувати **[warp.dev](https://www.warp.dev/)** — сучасний термінал із зручним копіпастом та підказками команд. Встановлюється як звичайний застосунок через `.dmg`, підходить навіть тим, хто вперше відкриває термінал на Mac.

Стандартний **Terminal.app** також підійде (Spotlight `⌘ + Space` → `Terminal`).

### Крок 1 — відкрити термінал

Запустіть **warp.dev** (або Terminal.app через Spotlight `⌘ + Space`).

### Крок 2 — перейти в папку з AutoPilot

```bash
cd ~/Downloads/AutoPilot
```

> **Лайфхак:** наберіть `cd ` (з пробілом), перетягніть папку з пілотом з Finder прямо у вікно терміналу — шлях підставиться сам. Потім натисніть Enter.

Перевірте, що ви у правильній папці:

```bash
ls
```

У виводі має бути файл `AutoPilot`, поруч — `AutoPilot.config`, `data/` та інші файли пілота.

### Крок 3 — зняти карантин (найважливіше!)

Це головний крок. Без нього Mac бачить файл як «документ» і не дає запустити:

```bash
sudo xattr -r -c ./AutoPilot
```

- Введіть пароль від Mac (при введенні символи **не відображаються** — це нормально, так задумано системою).
- `xattr -c` очищає розширені атрибути, включаючи `com.apple.quarantine`.
- `-r` застосовується рекурсивно (на випадок, якщо AutoPilot всередині папки з іншими файлами).

> Якщо `sudo` не спрацює — спробуйте без нього: `xattr -c ./AutoPilot`

### Крок 4 — зробити файл виконуваним

```bash
chmod +x ./AutoPilot
```

### Крок 5 — запустити

```bash
./AutoPilot
```

Якщо все зроблено правильно — AutoPilot запуститься у вікні терміналу, прочитає `AutoPilot.config` та почне роботу.

## Якщо щось пішло не так

### ❌ `zsh: killed` або `Killed: 9` при запуску

Карантин не знявся. Зніміть його **з усієї папки** рекурсивно:

```bash
sudo xattr -rc .
```

Крапка в кінці = поточна папка з усім вмістом.

### ❌ `Operation not permitted` на `chmod`

Виконайте з `sudo`:

```bash
sudo chmod +x ./AutoPilot
```

### ❌ `cannot be opened because the developer cannot be verified`

1. Відкрийте **System Settings → Privacy & Security**
2. Прокрутіть вниз до розділу **Security**
3. Знайдіть повідомлення про AutoPilot → натисніть кнопку **«Open Anyway»**
4. Поверніться в термінал і знову: `./AutoPilot`

### ❌ `No such file or directory`

Ви не в тій папці. Перевірте:

```bash
pwd     # покаже поточний шлях
ls      # покаже список файлів
```

Переконайтеся, що в `ls` є файл `AutoPilot`.

### ❌ `bad CPU type` або `Exec format error`

Завантажено версію для іншої архітектури. Перевірте свій Mac: `⌘ + Space` → **System Information** → розділ **Hardware** → **Chip**:

| Що у вас | Потрібна збірка |
|----------|-----------------|
| 🟢 Apple M1 / M2 / M3 / M4 | `darwin-arm64` |
| 🔵 Intel Core i5 / i7 / i9 | `darwin-x64` |

Завантажте правильну версію у постачальника.

### ❌ Mac бачить файл як «документ» у Finder

Це нормально — **запуск тільки через термінал** командою `./AutoPilot`. Подвійний клік по unsigned бінарнику у Finder на macOS принципово не працює. Іконка файлу у Finder не змінюється навіть після `chmod +x`.

## Коротка шпаргалка

Скопіюйте та виконайте по черзі (не забудьте підставити свій шлях):

```bash
cd ~/Downloads/AutoPilot            # ← підставте ваш шлях до папки
sudo xattr -r -c ./AutoPilot        # зняти карантин macOS
chmod +x ./AutoPilot                # зробити виконуваним
./AutoPilot                         # запустити
```

## Після першого запуску

Кроки 3 та 4 більше **не потрібні** — при подальших запусках достатньо:

```bash
cd ~/Downloads/AutoPilot
./AutoPilot
```

Атрибути карантину та права на виконання зберігаються доти, доки файл не буде повторно завантажено або переміщено в іншу папку.

---

## Після оновлення

Коли AutoPilot завантажить нову версію (див. [FAQ → Оновлення](/docs/ua/faq/#12--оновлення)) і ви розпакуєте архів із заміною файлу:

- **Права на виконання скидаються** — знову потрібен `chmod +x ./AutoPilot`
- **Карантин з'являється заново** (файл завантажено заново) — знову потрібен `sudo xattr -r -c ./AutoPilot`

Тобто після кожного оновлення повторіть кроки 3 та 4 з інструкції вище.

## FAQ

**Q: Чи треба робити це щоразу?**
A: Ні. Тільки при першому запуску або після оновлення AutoPilot (коли новий бінарник замінює старий).

**Q: Чи безпечно `sudo xattr -r -c`?**
A: Так. Команда не змінює вміст файлу, лише знімає позначку «завантажено з інтернету», яку macOS ставить автоматично на всі завантажені файли.

**Q: Чи можна запускати подвійним кліком, як звичайну програму?**
A: Ні. AutoPilot — це консольний застосунок, а не `.app`-пакет. Запуск тільки через термінал командою `./AutoPilot`.

**Q: Що робити, якщо термінал закрився, а AutoPilot ще працює?**
A: Процес продовжить роботу у фоні. Щоб подивитися логи, відкрийте папку `logs/` у папці пілота. Щоб зупинити — знайдіть процес через **Activity Monitor** або виконайте `killall AutoPilot` у новому терміналі.

**Q: Чи потрібен warp.dev, чи вистачить звичайного Terminal.app?**
A: Вистачить звичайного. warp.dev зручніший (копіпаст, підказки, історія), але це не обов'язково.

**Q: Mac питає пароль при `sudo` — який вводити?**
A: Пароль вашого користувача Mac (той самий, яким ви входите в систему). При введенні символи **не відображаються** — це нормально, просто наберіть пароль і натисніть Enter.

**Q: Gatekeeper не дає запустити навіть після `xattr -c`?**
A: Перезавантажте Mac і повторіть кроки. Якщо не допомогло — використайте обхідний шлях через System Settings → Privacy & Security → Open Anyway (див. вище).
