### 2. 🔐 2FA: як це працює

```mermaid
flowchart LR
    A["🔐 Підключення 2FA"] --> B["📱 Секрет збережено в таблицю"]
    B --> C["🔄 AutoPilot генерує TOTP"]
    C --> D["✅ Автоматичне введення коду"]

    style A fill:#9C27B0,color:#fff,stroke:none,rx:8
    style D fill:#4CAF50,color:#fff,stroke:none,rx:8
```

- При підключенні 2FA софт зберігає секретний ключ у стовпець `[2FA] totp_secret_code`
- Цей ключ можна імпортувати в будь-який додаток: Google Authenticator, Authy, 1Password та ін.
- AutoPilot автоматично генерує та вводить TOTP-коди за потреби
- Для ручної генерації коду із секретного ключа: [totp.danhersam.com](https://totp.danhersam.com/)

> 💾 **Бекап:** `totp_secret_code` додатково зберігається в логи профілю (папка `/logs`) — навіть якщо випадково очистите таблицю, секретний ключ завжди можна знайти в логах.

> ⚡ **Автопідключення:** дії `whitelist` та `withdraw` автоматично підключать 2FA, якщо він ще не встановлений. Окремо запускати `2fa` не потрібно.

> ⏰ **2FA коди не підходять?** TOTP-коди залежать від точного часу на комп'ютері. Якщо код відхиляється біржею — синхронізуйте годинник: **Windows:** Settings → Time & Language → Date & time → Sync now. **macOS:** System Settings → General → Date & Time → Set time and date automatically.

> 🌐 **Bybit: проблеми з 2FA?** Якщо код на пошту не приходить при підключенні Google Authenticator — вкажіть `bybitglobal` у стовпці `[BYBIT] domain` таблиці. У деяких регіонах `bybitglobal.com` працює стабільніше. Детальніше — [Bybit AutoPilot → Bybit Global домени](/docs/ua/bybit-autopilot/#-bybit-global-домени).

### 3. 🏎️ Режими швидкості (speed_mode)

| Режим | Курсор | Введення тексту | Коли використовувати |
|-------|--------|-----------------|---------------------|
| **⚡ FAST** | Миттєві переміщення | Швидке введення без помилок | Масові операції, коли швидкість важливіша |
| **🚗 MEDIUM** | Плавні рухи, середня швидкість | Людське введення з мінімальними помилками | Баланс між швидкістю та природністю |
| **🐢 SLOW** | Повільні рухи з рандомною траєкторією | Введення з помилками та виправленнями | Максимальна імітація людини |

**🖱️ Smart Cursor** (MEDIUM/SLOW): імітує людські рухи миші з рандомною траєкторією, прискореннями та уповільненнями.

**⌨️ Human Typing** (MEDIUM/SLOW): випадкові затримки між символами, випадкові друкарські помилки з автовиправленням — як справжня людина.

> 💡 **Рекомендація:** для реєстрації та логіну достатньо FAST. Для трейдингу та дій на сторінках з антибот-захистом — MEDIUM.

### 4. 🧩 Проксі та капча

**Стовпець `[PROXY] proxy`** — проксі профілю у форматі `ip:port:user:password`

> ✅ **Заповнювати не потрібно** — AutoPilot автоматично бере проксі з вашого профілю AdsPower / Dolphin / Vision / Afina на льоту. Проксі передається в сервіс вирішення капчі, щоб капча була вирішена з IP профілю. Все відбувається автоматично.

#### 🧩 Підтримувані провайдери капчі

AutoPilot підтримує **4 провайдери** вирішення капчі. Обирається параметром `captcha_provider` у конфігу:

| Провайдер | `captcha_provider` | Тип | Сайт |
|-----------|:---:|-----|------|
| **CapSolver** ⭐ | `capsolver` | Token (GeeTest v4) | [capsolver.com](https://www.capsolver.com/) |
| **CapMonster** | `capmonster` | Token (GeeTest v4) | [capmonster.cloud](https://capmonster.cloud/) |
| **2Captcha** | `2captcha` | Token (GeeTest v4) | [2captcha.com](https://2captcha.com/) |
| **CapGuru** | `capguru` | Visual (перетягування) | [cap.guru](https://cap.guru/) |

> ⭐ **Рекомендовано: CapSolver** — найстабільніший та найшвидший на всіх біржах (Bybit, MEXC, Bitget). Token-based рішення GeeTest v4, API-first підхід, підтримка нових типів капчі з'являється одразу після релізу.

> 🖼️ **CapGuru** — працює тільки з **візуальною капчею** (перетягування слайдера). Не підтримує token-based GeeTest v4. Використовується як fallback там, де token не працює.

**Приклад конфігурації:**

```
captcha_key=CAP-ВАШ_КЛЮЧ_ТУТ
```

> 💡 **Автовизначення провайдера:** AutoPilot автоматично визначає провайдера капчі за форматом ключа. Ключ починається з `CAP-` — значить CapSolver. Параметр `captcha_provider` можна не вказувати — достатньо вставити ключ у `captcha_key`, і AutoPilot сам зрозуміє, який це сервіс.

#### 🔑 Як налаштувати CapSolver (покроково)

1. Зареєструйтесь на [capsolver.com](https://www.capsolver.com/)
2. Перейдіть у **Billing → Top Up** та поповніть баланс від **$10** криптовалютою
3. Скопіюйте **API Key** з особистого кабінету (Overview → API Key)
4. Вставте ключ у `AutoPilot.config`:
   ```
   captcha_key=CAP-ваш_ключ
   ```
5. Готово — AutoPilot автоматично визначить, що це CapSolver

> ⚠️ **НЕ купуйте пакети на сторінці Market!** Там продаються підписки на конкретні типи капчі (reCAPTCHA v2 50K за $36, reCAPTCHA v3 тощо) — **AutoPilot їх не використовує**. AutoPilot працює з GeeTest v4 (token-based), для цього потрібен лише **баланс на акаунті**, а не підписка на пакет. Поповніть баланс від $10 через Billing → Top Up — і все працюватиме.

> 💰 **Вартість:** ~$1 за 1000 рішень GeeTest. Поповнення $10 вистачає надовго.

> 🔧 **Помилки капчі:** якщо капча постійно не вирішується — перевірте валідність ключа `captcha_key` в конфігу та баланс акаунта на сайті провайдера. Якщо проблема тільки на одній біржі — спробуйте перемкнути тип рішення — з token-based на visual (наприклад, `capsolver` → `capguru`).

> 📁 **Excel повністю закритий?** Таблиця повинна бути повністю закрита під час роботи AutoPilot:
> - Натискання ✕ в Excel **часто недостатньо** — Windows залишає `EXCEL.EXE` у фоні (через OneDrive sync, AutoSave, декілька вікон). Перевірте Task Manager (`Ctrl+Shift+Esc` → вкладка Processes → знайдіть `EXCEL.EXE` → End task)
> - Швидка альтернатива — через командний рядок: `taskkill /IM EXCEL.EXE /F`
> - Якщо `EXCEL.EXE` висить у фоні, AutoPilot читає застарілі дані або взагалі не може відкрити файл. Класичний симптом: «дані з таблиці не підтягуються в консоль, допомагає лише перезавантаження ПК»

> ☁️ **AutoPilot повинен знаходитись у локальній папці**, а не в OneDrive / Dropbox / iCloud Drive / Google Drive / будь-якій іншій хмарній синхронізації. Cloud-sync тримає `AutoPilot_table.xlsx` відкритим у фоновому режимі для синхронізації та блокує читання/запис AutoPilot'ом.
>
> **Куди покласти:**
> - `C:\AutoPilot\` або `D:\AutoPilot\` — будь-який локальний диск
> - Або найзручніше — на робочому столі: `\Desktop\AutoPilot\` (на Windows 11 переконайтеся, що Desktop не синхронізується OneDrive — іконка OneDrive у треї → Settings → Backup → вимкнути Desktop)

### 6.1 🤖 Розумна ф'ючерсна торгівля (futures_smart)

ACTION: `futures_smart` — автоматична торгівля ф'ючерсами з аналізом ринку, лімітними post-only ордерами та управлінням позиціями.

```mermaid
flowchart TD
    A["📊 Аналіз ринку"] --> B{"🧠 Сигнал"}
    B -->|Long ↑| C["📝 Лімітний ордер Buy"]
    B -->|Short ↓| D["📝 Лімітний ордер Sell"]
    C --> E{"⏳ Заповнений?"}
    D --> E
    E -->|Ні, таймаут| A
    E -->|Так| F["📈 Утримання позиції"]
    F --> G["🔄 Закриття по маркету"]
    G --> H{"🎯 Цільовий обсяг?"}
    H -->|Ні| I["💤 Пауза"]
    I --> A
    H -->|Так| J["✅ Готово"]

    style A fill:#2196F3,color:#fff,stroke:none,rx:8
    style J fill:#4CAF50,color:#fff,stroke:none,rx:8
    style C fill:#009688,color:#fff,stroke:none,rx:8
    style D fill:#f44336,color:#fff,stroke:none,rx:8
```

**💰 Комісії:** post-only лімітні ордери = 0.02% (maker), закриття по маркету = 0.055% (taker). Разом ~0.075% за цикл — **на 32% дешевше** ніж маркет ордери (0.11%).

**📊 Стовпці таблиці:**

| Стовпець | Опис | Приклад |
|----------|------|---------|
| `[TRADING] trading_coin` | Монета для ф'ючерсів | `BTC` |
| `[TRADING] trading_amount` | Розмір позиції за одну ітерацію (з урахуванням плеча) в USDT | `1000` |
| `[TRADING] trading_cycles` | Кількість торгових циклів | `15` |

> 🔑 **Важливо:** `trading_amount` — це розмір позиції **з урахуванням плеча**, а не сума реально використовуваного балансу. Реальний баланс на одну угоду = `trading_amount / leverage`.

**📐 Формула:**
- **Реальний баланс за угоду** = `trading_amount ÷ плече`
- **Обсяг за цикл** = `trading_amount × 2` (відкриття + закриття)
- **Цільовий обсяг** = `trading_amount × cycles × 2`

**Приклад:** `trading_amount=1000`, `leverage=10`, `cycles=15`
- Реальний баланс за угоду: 1000 ÷ 10 = **100 USDT**
- Обсяг за один цикл: 1000 × 2 = **2000 USDT**
- Цільовий обсяг: 1000 × 15 × 2 = **30 000 USDT**

**⚙️ Параметри конфігу:**

| Параметр | Опис | За замовчуванням |
|----------|------|:----------------:|
| `leverage` | 📊 Кредитне плече | `10` |
| `limit_futures_position_hold_interval` | ⏳ Час утримання позиції, сек (мін,макс) | `50,240` |
| `limit_futures_iteration_wait_interval` | 💤 Пауза між циклами, сек (мін,макс) | `60,180` |
| `limit_futures_order_timeout` | ⏱️ Макс. очікування заповнення лімітного ордера, сек | `30` |
| `limit_futures_order_check_interval` | 🔍 Інтервал перевірки виконання ордера, сек | `5` |
| `limit_futures_price_offset_ticks` | 📐 Відступ у тіках від найкращої bid/ask ціни | `2` |
| `limit_futures_direction_algorithm` | 🧠 Алгоритм визначення напрямку | `inverse_simple` |
| `limit_futures_order_deviation` | 🎲 Випадкове відхилення розміру позиції, % (0 = вимк) | `0` |

**🧠 Алгоритми визначення напрямку:**
- `simple` — слідує за трендом: аналіз останніх 5 свічок + ціна vs MA5 (зростання → Long, падіння → Short)
- `inverse_simple` — контр-тренд: інвертує сигнал simple (зростання → Short, падіння → Long)
- `advanced` — комплексний аналіз: EMA 8/21 кросовер + RSI 14 + Volume Spike (1.7x від середнього) + Funding Rate

> ⚠️ **Ризики:** ф'ючерсна торгівля з плечем пов'язана з ризиком ліквідації. Використовуйте помірне плече та невеликі суми для набиття обсягу.

> 💡 **Порада:** якщо ордер не заповнюється протягом `limit_futures_order_timeout` — сигнал перераховується заново. Збільшіть `limit_futures_price_offset_ticks` для швидшого заповнення (але з менш вигідною ціною).

**✨ Можливості:**
- 📱 Автоматична реєстрація в пазлі та його менеджмент
- 📱 Миттєве виконання та клейм соцзавдань напряму через запити
- 📱 Автоматичний прогін щоденних трейдинг-завдань та чек-інів
- 📱 Можна вказати декілька пазлів одразу — система сама все зробить
- 📱 Не буде витрачати спроби на торгівлю, якщо денний чек-ін вже закритий
- 📱 Ведеться таблиця з ключовими даними: скільки днів завершено / всього, плюс відмітка про успішне проходження пазла цілком

**📊 Стовпець таблиці:**

| Стовпець | Опис | Приклад |
|----------|------|---------|
| `[PUZZLE] event_code` | Код пазла (або декілька через кому) | `0768558741987` |

> 📌 **Тип стовпця:** зробіть стовпець `[PUZZLE] event_code` **текстовим** (не числовим), інакше Excel може обрізати довгі коди.

**🔗 Де взяти код пазла:**

Код знаходиться в посиланні пазла — параметр `activityCode`:
```
https://www.bybit.com/en/trade/spot/puzzle-hunt/detail?activityCode=0768558741987
                                                                      ^^^^^^^^^^^^^ — це код
```

**📋 Менеджмент декількох пазлів:**

Щоб виконати декілька пазлів за раз — вкажіть коди через кому:
```
8675373977840,0768558741987,1234567890123
```
Система обробить кожний пазл послідовно: соцзавдання → торгівля (батчем) → чек-іни.

**⚙️ Як працює всередині:**

1. **Перевірка ліміту** — перед будь-якою роботою перевіряється, чи доступний денний чек-ін. Якщо ліміт вичерпано — пазл пропускається цілком (торгівля не витрачається даремно)
2. **Реєстрація** — автоматична реєстрація учасника в пазлі
3. **Соцзавдання** — виконання всіх соціальних завдань (підписки, ретвіти) через прямі API-запити + клейм нагород (фрагменти пазла)
4. **Торгівля** — якщо сьогодні ще не торгували: купівля + продаж монети зі списку пазла (~5 USDT обсяг). Якщо вже торгували — пропускає
5. **Очікування** — пауза 3-4 хвилини, щоб сервер Bybit зафіксував обсяг торгівлі
6. **Чек-ін** — денна відмітка + отримання фрагментів пазла

**📈 Статуси в таблиці:**

| Статус | Значення |
|--------|----------|
| `DONE` | ✅ Пазл повністю завершено (всі дні пройдено) |
| `5/10` | 🔄 Прогрес: 5 днів з 10 завершено |
| `ENDED` | ⏰ Пазл закінчився (активність завершена на Bybit) |
| `FAIL` | ❌ Помилка (деталі в логах) |

При декількох пазлах статус комбінується: `[CODE1] DONE, [CODE2] 3/10`

> 💡 **Порада:** запускайте `puzzle_hunt` щодня — софт сам визначить, що вже зроблено, і виконає тільки відсутні кроки.

> ⚡ **puzzle_social:** якщо потрібно виконати тільки соцзавдання без торгівлі — використовуйте ACTION `puzzle_social`.

### 6.3 💰 Bybit Earn — стейкінг USDT (earn / unearn)

ACTION: `earn` — відправити USDT в Flexible Savings пул (автоматичний стейкінг).
ACTION: `unearn` (або `unstake`) — забрати USDT з пулу назад на Funding-рахунок.

```mermaid
flowchart LR
    subgraph earn ["earn — стейкінг"]
        direction LR
        A1["💳 Trading рахунок"] --> A2["💳 Funding рахунок"]
        A2 --> A3["🏦 Flexible Savings"]
    end

    subgraph unearn ["unearn — виведення зі стейкінгу"]
        direction LR
        B1["🏦 Flexible Savings"] --> B2["💳 Funding рахунок"]
    end

    style A3 fill:#4CAF50,color:#fff,stroke:none,rx:8
    style B2 fill:#2196F3,color:#fff,stroke:none,rx:8
```

**⚙️ Як працює:**

**`earn` (стейкінг):**
1. Перевірка KYC та авторизація на Bybit
2. Переведення всіх USDT з Trading-рахунку на Funding-рахунок (`transferAllToFunding`)
3. Отримання балансу доступних USDT на Funding-рахунку
4. Відправка в пул Flexible Savings: preview → pay-order → confirm-order
5. Статус у таблиці: `[STAKE] SUCCESS - stake 150.25`

**`unearn` (виведення зі стейкінгу):**
1. Перевірка KYC та авторизація на Bybit
2. Отримання поточного балансу в Flexible Savings
3. Виведення з пулу на Funding-рахунок: preview → pay-order → confirm-order
4. Статус у таблиці: `[STAKE] SUCCESS - unstake 150.25`

**📊 Стовпці таблиці:**

| Стовпець | Опис | Приклад |
|----------|------|---------|
| `[WITHDRAW] withdraw_amount` | Фіксована сума USDT (необов'язково) | `100` |

> 💡 **За замовчуванням:** якщо `withdraw_amount` не заповнено — `earn` відправить **весь** доступний баланс USDT в пул, а `unearn` забере **все** з пулу.

> 📌 **Фіксована сума:** вкажіть число в `withdraw_amount` для стейкінгу/виведення конкретної суми. Якщо запитана сума перевищує доступний баланс — буде використано весь доступний баланс.

> 🏦 **Flexible Savings:** це гнучкий стейкінг — кошти можна забрати в будь-який момент, відсотки нараховуються щодня. Виведення миттєве.

> ⚡ **Автоматичний трансфер:** при `earn` кошти автоматично переводяться з Trading-рахунку на Funding перед стейкінгом — не потрібно робити це вручну.

> 🔄 **Псевдоніми:** `stake` = `earn`, `unstake` = `unearn` — ідентичні дії, використовуйте будь-яке.

### 6.4 🛒 Варіанти торгівлі (buy, limit, futures)

AutoPilot пропонує декілька торгових дій для різних сценаріїв:

| Дія | Тип | Опис |
|-----|-----|------|
| `trading` | Market | Цикли купівля-продаж маркет ордерами. Обсяг = `цикли × сума × 2` |
| `trading_limit` | Limit | Те саме що `trading`, але лімітними ордерами для кращих цін |
| `buy` | Market | Разова купівля конкретного активу |
| `limit_buy` | Limit | Разова лімітна купівля |
| `limit` | Limit | Випадкова лімітна торгівля для природного обсягу |
| `sell` | Market | Продаж усіх активів за USDT |
| `limit_sell` | Limit | Продаж на Funding-рахунку лімітними ордерами |
| `futures` | Market | Ф'ючерси з плечем маркет ордерами |
| `futures_smart` | Limit | Розумні ф'ючерси з post-only ордерами (на 32% дешевше) |

> 💡 **Коли що використовувати:** `trading` для швидкого обсягу, `trading_limit` для кращих цін, `futures_smart` для найнижчих комісій. `buy` / `limit_buy` для разових покупок.

> 📊 **Стовпці:** всі торгові дії використовують `[TRADING] trading_coin`, `[TRADING] trading_amount`. Циклічні дії також використовують `[TRADING] trading_cycles`.

### 7.1. 🔑 Мастер-гаманець — Авто-whitelist, $0 комісії, невідстежуваний вивід

Виводьте з сотень акаунтів **не вводячи жодної адреси вручну**. AutoPilot створює унікальний гаманець для кожного акаунта, сам додає його в whitelist, виводить з **нульовою комісією** і збирає все на ваш мастер-гаманець. Кожен акаунт виводить на **свою адресу** — зв'язати їх між собою неможливо.

```mermaid
flowchart LR
    A["🔐 setup_aptos_hd"] --> B["🏷️ whitelist_aptos_hd"]
    B --> C["💳 withdraw"]
    C --> D["📥 Distribute: збір"]

    style A fill:#9C27B0,color:#fff,stroke:none,rx:8
    style B fill:#FF5722,color:#fff,stroke:none,rx:8
    style C fill:#FF9800,color:#fff,stroke:none,rx:8
    style D fill:#4CAF50,color:#fff,stroke:none,rx:8
```

**Що ви отримуєте:**
- 🏷️ **Автоматичні вайтлісти** — не потрібно створювати адреси чи заповнювати таблицю. AutoPilot створює унікальну адресу для кожного акаунта і додає її в whitelist на Bybit з повним підтвердженням email/2FA
- 🕵️ **Невідстежувані виведення** — кожен акаунт виводить на свою унікальну адресу. Спільної адреси немає = зв'язати ваші акаунти неможливо
- 💰 **$0 комісія за виведення** — Bybit не бере плату за виведення в мережі Aptos (проти $1-2 за акаунт на ERC20/TRC20)
- 📥 **Збір в один клік** — після виведення збираєте всі кошти на мастер-гаманець за секунди
- ⚡ **Миттєві переводи** — кошти приходять за ~1 секунду

**Як це працює (3 дії):**

1. **`setup_aptos_hd`** — запускається один раз. Створює ваш мастер-гаманець і дає секретну фразу (12 слів). Одна фраза = необмежена кількість адрес.

2. **`whitelist_aptos_hd`** — запускається на всіх профілях. AutoPilot сам:
   - Створює унікальну Aptos-адресу для кожного акаунта
   - Заходить на Bybit, додає її в whitelist, підтверджує email/2FA — все автоматично
   - Зберігає все в таблицю

3. **`withdraw`** — виводить USDT на whitelisted адресу кожного акаунта. Комісія = **$0**.

Після виведення — відкрийте вкладку **Distribute** і зберіть все на один гаманець одним кліком.

| Параметр конфігурації | Опис | За замовч. |
|-----------|-------------|:-------:|
| `aptos_hd_enabled` | Увімкнути систему мастер-гаманця | `NO` |
| `aptos_hd_auto_assign` | Автопризначення адреси новим профілям | `YES` |
| `aptos_distribution_master_key` | Приватний ключ мастер-гаманця (hex) | — |
| `aptos_distribution_master_index` | Індекс мастер-гаманця | `0` |
| `aptos_distribution_amount` | Сума USDT на кожну роздачу | `0.5` |

> 💡 **Приклад:** у вас старі акаунти Bybit із залишками. Замість ручного створення адрес, введення їх по одній та збору з кожного гаманця — запустіть 3 дії і все на вашому мастер-гаманці. Нуль ручної роботи, нуль комісій, нуль зв'язків між акаунтами.

> 🔁 **Можна перезапускати:** якщо акаунт вже має адресу, `whitelist_aptos_hd` використає ту саму — дублікатів не буде.

### 16. 🍎 Запуск на macOS

> 📥 **Спочатку завантажте збірку для macOS** з AutoPilot Chat (Telegram) — гілка **Updates Bybit / MEXC / Bitget** залежно від біржі. Детальніше — [секція 12: Оновлення](#12--оновлення).

На macOS AutoPilot **не запускається подвійним кліком** з Finder — система блокує непідписані бінарні файли. Запуск тільки через термінал, і тільки після зняття карантину Gatekeeper.

**Що ви побачите:**
- Файл `AutoPilot` у Finder відображається як «документ», а не програма
- Помилки: `cannot be opened because the developer cannot be verified`, `killed: 9`, `Operation not permitted`

**Коротка шпаргалка** (виконайте по черзі в терміналі — рекомендуємо [warp.dev](https://www.warp.dev/)):

```bash
cd ~/Downloads/AutoPilot            # ваш шлях до папки
sudo xattr -r -c ./AutoPilot        # зняти карантин macOS
chmod +x ./AutoPilot                # зробити виконуваним
./AutoPilot                         # запустити
```

> 🍎 **Повний гайд** (troubleshooting помилок `killed: 9` / Gatekeeper, Apple Silicon vs Intel, що робити після оновлення, FAQ): **[Запуск AutoPilot на macOS →](/docs/ua/macos-launch/)**

> 💡 **Після першого запуску** кроки `xattr` та `chmod` більше не потрібні — при подальших запусках достатньо `cd` у папку та `./AutoPilot`. Після кожного оновлення повторіть ці кроки (новий бінарник знову приходить з карантином).
