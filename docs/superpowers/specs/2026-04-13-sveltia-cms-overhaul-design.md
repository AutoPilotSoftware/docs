# Sveltia CMS + Starlight GitBook-like Overhaul

**Date:** 2026-04-13
**Status:** Approved
**Goal:** Transform the autopilot-docs Starlight site into a GitBook-like documentation platform where admins can create, edit, delete articles with rich content blocks, images, and hierarchical navigation — all through Sveltia CMS's browser-based editor, with zero infrastructure beyond GitHub Pages.

## Reference

- Target UX: [docs.astrum.foundation](https://docs.astrum.foundation/) (GitBook)
- Current site: [autopilotsoftware.github.io/docs](https://autopilotsoftware.github.io/docs)
- Stack: Astro 5.6 + Starlight 0.37 + Sveltia CMS (already partially set up)
- Deploy: GitHub Actions → GitHub Pages (unchanged)

## Current State

- Sveltia CMS at `/admin` with `create: false, delete: false`
- Flat content structure — all articles in `src/content/docs/` root
- Manual sidebar in `astro.config.mjs` (9 hardcoded entries)
- Plain `.md` files (not MDX except `index.mdx`)
- No custom editor components (no rich blocks)
- Basic `custom.css` (image centering only)
- i18n: ru (root), en, ua — working with `multiple_folders`

## Architecture Overview

```
Admin (browser)
  ↓ Sveltia CMS (client-side JS)
  ↓ GitHub API (PAT or OAuth)
  ↓ Commits to repo
  ↓ GitHub Actions builds Astro/Starlight
  ↓ Deploys to GitHub Pages
User reads static site
```

No backend, no database, no server. CMS runs entirely in the browser.

### 2.2 Custom Editor Components

Register in `public/admin/index.html` before the CMS script:

```html
<script>
  // Wait for CMS to load, then register components
  window.CMS_MANUAL_INIT = true;

  document.addEventListener('DOMContentLoaded', () => {
    const script = document.createElement('script');
    script.src = 'https://unpkg.com/@sveltia/cms/dist/sveltia-cms.js';
    script.type = 'module';
    script.onload = () => {
      // Starlight Aside / Callout block
      CMS.registerEditorComponent({
        id: 'aside',
        label: 'Callout',
        icon: 'campaign',
        fields: [
          {
            name: 'type',
            label: 'Type',
            widget: 'select',
            options: ['note', 'tip', 'caution', 'danger'],
            default: 'note',
          },
          { name: 'title', label: 'Custom Title', widget: 'string', required: false },
          { name: 'content', label: 'Content', widget: 'text' },
        ],
        pattern: /^:::(\w+)(?:\[(.+?)\])?\n([\s\S]+?)\n:::/m,
        fromBlock: (match) => ({
          type: match[1],
          title: match[2] || '',
          content: match[3]?.trim(),
        }),
        toBlock: (data) => {
          const header = data.title
            ? `:::${data.type}[${data.title}]`
            : `:::${data.type}`;
          return `${header}\n${data.content}\n:::`;
        },
        toPreview: (data) =>
          `<div style="border-left:4px solid #3b82f6;padding:8px 12px;margin:8px 0;background:#f0f4ff;border-radius:4px"><strong>${data.type}${data.title ? ': ' + data.title : ''}</strong><br/>${data.content}</div>`,
      });

      CMS.init();
    };
    document.body.appendChild(script);
  });
</script>
```

Produces Starlight-native aside syntax:
```markdown
:::tip[Pro tip]
Use FAST mode for maximum throughput
:::
```

## Phase 4: Auth & Deployment

### 4.1 PAT-Based Auth (Phase 1)

1. Create GitHub bot account (e.g., `AutoPilotAdmin`) or use existing
2. Add bot as collaborator with write access to `AutoPilotSoftware/docs`
3. Create fine-grained PAT: Contents → Read and Write, scoped to `docs` repo
4. Admin logs in at `/docs/admin/` with PAT — commits appear as bot account

### 4.2 OAuth Multi-Admin (Phase 2, optional)

If multiple admins needed later:
1. Deploy `sveltia-cms-auth` Cloudflare Worker (free)
2. Create GitHub OAuth App
3. Add `base_url` in `config.yml` pointing to Worker
4. Admins log in with GitHub account (one-click)
